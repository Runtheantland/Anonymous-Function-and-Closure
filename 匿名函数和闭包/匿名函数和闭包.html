一、匿名函数

匿名函数：没有名字的函数

1正常函数


function box(){
  return "我是正常函数";
};
alert(box());

2.匿名函数


  function(){
    return "我是匿名函数";
  }
  因为函数主要是执行和调用
  单独的匿名函数无法运行，就算运行了也无法调用

3.匿名函数赋值给变量


 var main=function(){
   return "我是匿名函数";
 }
alert(main());

4.通过表达式自我执行匿名函数


  (function(){

  })();

4.自我执行匿名函数传参


  (function(num){

  })(100);

二、闭包

套路太深

可以读取函数内部的变量

让这些变量始终保存在内存中

1.函数里放一个匿名函数


function box(){
  return function(){    //闭包
    return "我是匿名函数";
  }
};
alert(box()());
var b=box();//灵活
alert(b());
闭包：有权访问另一个函数作用域中的函数，创建闭包的常见方式就算在一个函数内部创建另一个函数，通过另一个函数访问这个函数的局部变量

2通过闭包返回局部变量.


function box(){
  var age=100;
  return function(){ //闭包
    return age;
  }
};
alert(box()());
var b=box();//灵活
alert(b());
使用闭包有一个优点，也是缺点，就算可以把局部变量驻留在内存中，可以避免使用全局变量(全局变量污染导致程序不可预测性，每个模块都可以调用必将引来灾难，
所以推荐使用私有的封装的局部变量)命名冲突，各个模块去调用全局变量各种异常的问题全局变量污染等一般要模块化，私有化封装起来。

4.通过局部变量进行累加



function box(){
  var age=100;
  age++;
  return age;
}
alert(box());//101
alert(box());//101

5.通过匿名函数实现局部变量驻留在内存中进行累加


function box(){
  var age=100;
  return function(){
    age++;
    return age;
  };
}
var b=box();
alert(b());//101
alert(b());//102
b=null;//解除引用
闭包里作用域返回的是局部变量不会被立刻回收，所以可能会占用更多的内存，过度的使用闭包会导致性能下降，有必要时才使用

11.内存泄漏


  window.onload = function(){
      var ele = document.getElementById("id");
      ele.onclick = function(){
          alert(ele.id);
      }
  }
  这段代码为什么会造成内存泄露？

  ele.onclick = function(){
          alert(ele.id);
      }
  执行这段代码的时候，将匿名函数对象赋值给ele的onclick属性；然后匿名函数内部又引用了ele对象，存在循环引用，所以不能被回收。

  解决方法：

  window.onload = function(){
      var ele = document.getElementById("id");
      var id = ele.id; //解除循环引用
      ele.onclick = function(){
          alert(id);
      }
      ele = null; // 将闭包引用的外部函数中活动对象清除
  }

三.模仿块级作用域(javascript没有块级作用域的概念)

比如说很多块级部分for if语句讲里面的东西封装起来，将里面的变量私有化，模块化(保护数据，防止内存泄漏，安全性和可控性更高)


function box(count){
  for(var i=0;i< count;i++){  //块级作用域(js本身没有这个东西)
    //执行了一堆操作
  };
  //var i;    //就算重新声明，也不会印象之前初始化的数据
  alert(i);//5
}
box(5);
说明js没有块级语句的作用域，if().for(){}....有的话出了这个就应该被销毁了，可是就算重新声明一遍都不会影响它的值
js更不会提醒你是否多次声明了一个变量；有这种情况只会对后面的声明装作看不见（初始化除外），使用模仿块级作用域就可以了


使用块级作用域(私有作用域)
function box(count){
  (function(){//包含自我执行的匿名函数，就可以实现私有作用域
  for(var i=0;i< count;i++){

  }
  //这里用了a,b,c等变量
  })()//被销毁了
  //这里可以继续使用a,b,c等变量
 alert(i);
}
box(5);
